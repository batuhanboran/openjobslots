# OpenJobSlots v1.6 Quality Hardening Plan

This document is the baseline map for queued v1.6 hardening work. It is intentionally scoped to planning, test harnesses, and safe verification. It does not deploy production.

## Current Architecture Summary

- Public app: React Native Web / Expo frontend served by the Node API container in production.
- API server: `server/index.js` exposes public search/status routes and admin/control routes.
- Ingestion worker: `server/ingestion/worker.js` runs separately from the API container.
- Production compose services: `openjobslots-app`, `openjobslots-worker`, `openjobslots-postgres`, `openjobslots-meilisearch`.
- Active production storage in compose: Postgres via `OPENJOBSLOTS_DB_BACKEND=postgres`.
- Active production search in compose: Meilisearch via `OPENJOBSLOTS_SEARCH_BACKEND=meili`.
- Queue/control model in compose: Postgres sync-control state; pg-boss code exists but is not the active queue path.
- SQLite remains available for local fallback and isolated Playwright/API test DBs.
- Meilisearch is a derived public search index. Postgres remains source of truth.
- Current package version at this baseline: `1.5.30`.
- ATS configuration count from code: 60 keys, with `dayforcehcm` disabled until certification.
- Strict raw parser fixture coverage from code: 16 ATS keys.
- Normalized or parser fixture coverage from code: 21 ATS keys.

## Remaining Issue Areas

1. Search relevance and Meili/search parameter correctness.
   - Risk: Meili settings, synonyms, fallback SQL, hydration, and filters can disagree.
   - Existing mitigation: `scripts/check-search-parity.js` compares public API, active search, Postgres, and Meili for representative queries.

2. ATS parser correctness and adapter certification.
   - Risk: incomplete geo/date/remote/source-id normalization causes correct jobs to disappear under filters.
   - Existing mitigation: parser fixtures and certification metadata in `server/ingestion` and `docs/ats-*`.

3. Ingestion sync/cache production hardening.
   - Risk: worker writes, status, queue state, cache invalidation, and sync controls can drift.
   - Existing mitigation: separate worker container, Postgres sync-control paths, and worker concurrency tests.

4. UI/UX bugs found by automation.
   - Risk: search keyboard behavior, pagination, filter chips, and stale-result states can regress without backend failures.
   - Existing mitigation: Playwright E2E tests in `tests/e2e`.

5. Data quality diagnostics and explainability.
   - Risk: parser attention counts are visible but may not explain root cause enough to drive fixes.
   - Existing mitigation: admin/parser diagnostics, `audit:ats-quality`, and field-quality docs.

## Existing Tests And Gaps

Existing npm scripts discovered from `package.json`:

- `test:backend`: syntax checks, backend unit tests, reindex checks, parity tool tests, backfill tests, search corpus, and parser tests.
- `test:api`: Playwright API compatibility tests against an isolated test stack.
- `test:parsers`: ingestion/parser fixture tests.
- `test:e2e`: Playwright desktop/mobile public UI flow tests against an isolated test stack.
- `test:search-corpus`: local search corpus behavior.
- `search:parity`: live-like API/Postgres/Meili parity command.
- `reindex:meili`: Meili reindex and check mode.
- `audit:ats-quality`: ATS field-quality audit.

New baseline entry point:

- `quality:gate`: discovers scripts and runs safe tests in order: backend, API, parsers, E2E.

Known gaps:

- The local 1000-case corpus is useful but not a replacement for production parity checks.
- Large 10,000+ title-intent parity work is planned but not committed as a huge static dataset.
- Parser tests prove saved fixture behavior, not every live tenant variation.
- E2E tests validate public flows, not admin diagnostic workflows exhaustively.
- Meili count comparisons on broad searches can be capped by Meili result limits; parity must explain differences instead of relying on counts alone.

## Known Risky Files And Modules

- `server/index.js`: large legacy API and collector surface. Public reads, admin controls, and legacy sync code share one file.
- `server/backends/postgresStore.js`: search, hydration, filters, status, sync-control, and diagnostics.
- `server/search/meili.js`: Meili settings, document shape, query behavior, and reindex assumptions.
- `scripts/check-search-parity.js`: production parity logic and case definitions.
- `scripts/reindex-meili-from-postgres.js`: index safety checks and reindex write path.
- `server/ingestion/adapters.js`: adapter dispatch and parser normalization.
- `server/ingestion/posting.js`: canonical normalized posting rules.
- `server/ingestion/adapter-metadata.js`: configured ATS catalog and fixture status.
- `server/ingestion/ats-certification.js`: certification records and field-decision logic.
- `server/ingestion/worker.js`: sync scheduling, write behavior, and cache/update flow.
- `scripts/backfill-posting-normalization.js`: dry-run/write backfill logic.
- `scripts/backfill-ats-detail-pages.js`: sampled detail-page repair.
- `App.js` and `src/api.js`: search UX, public error handling, sync UI state, and protected route boundaries.
- `playwright.config.js` and `scripts/test/e2e-stack.js`: isolated test stack behavior and artifact capture.

## Task Checklist For Queued Prompts 2 Through 6

### Prompt 2 - Search Relevance And Parameter Correctness

- Expand `search:parity` cases for title plus country, title plus region, and title plus remote mode.
- Add hard-negative cases where the title matches but country or remote mode must reject the row.
- Verify Meili synonyms, `matchingStrategy`, sort behavior, and hydration underfill behavior.
- Confirm Postgres fallback and Meili path return compatible public shapes.
- Add tests before changing runtime behavior.

### Prompt 3 - ATS Parser Correctness And Certification

- Rank ATS by live field gaps using `audit:ats-quality`.
- Select a small high-impact batch.
- Add raw fixtures, expected normalized fixtures, and failure fixtures.
- Improve parser normalization only where source evidence supports it.
- Update `docs/ats-adapter-matrix.md` and `docs/ats-source-certification.md`.
- Do not mark an ATS certified without passing raw fixture-backed tests.

### Prompt 4 - Ingestion Sync And Cache Hardening

- Audit worker write paths and public read paths for accidental writes.
- Confirm sync start/stop state survives app and worker restarts.
- Verify cache invalidation after ingestion writes and backfills.
- Add tests for worker interruption, retry/backoff, and parser attention grouping.
- Keep API search usable while worker writes.

### Prompt 5 - UI/UX Automation Bugs

- Use Playwright artifacts from failed runs before changing UI.
- Fix only reproducible public search UX bugs.
- Preserve keyboard-first behavior: Enter searches, `/` and Ctrl/Cmd+K focus search, Esc clears.
- Confirm no protected/admin routes are requested from the public search page.
- Verify desktop and mobile, including scroll, pagination, filters, suggestions, and stale-result banners.

### Prompt 6 - Data Quality Diagnostics And Explainability

- Improve `/admin/parsers` or equivalent diagnostics with grouped ATS reasons, last sample, and next action.
- Make parser attention actionable without exposing private/raw payloads publicly.
- Add field-gap summaries by ATS for country, region, city, remote, date, source id, department, employment type, and description.
- Add a documented path from diagnostic reason to parser fixture or backfill action.
- Keep public UI diagnostics compact and non-sensitive.

## Safe Deploy Checklist For Final Prompt

- Confirm branch is merged or intentionally deployed from the expected commit.
- Run `npm run quality:gate`.
- Run `npm run search:parity` against the intended environment if Meili/Postgres behavior changed.
- Run `npm run reindex:meili -- --check` before any Meili write/rebuild.
- Confirm no production DB files changed during local test runs.
- Confirm compose services are only the expected OpenJobSlots services.
- Confirm `/health`, `/sync/status`, `/ingestion/status`, `/postings?search=t%C3%BCrkiye`, and `/postings?search=turkish%20jobs`.
- Confirm public UI shows no raw DB, stack trace, token, or admin errors.
- Confirm rollback path is known:
  - for search issues, switch `OPENJOBSLOTS_SEARCH_BACKEND=postgres`;
  - for app issues, redeploy the previous known-good commit;
  - for Meili issues, rebuild index from Postgres after check mode passes.
- Do not run production write backfills or full replace reindex without a dry run and explicit approval in the deploy prompt.
